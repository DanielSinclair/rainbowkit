# 模態鉤子

## 通過鉤子以編程方式開啟模態

提供了以下鉤子，允許在您的應用程序中的任何地方以編程方式打開模態。

- `useConnectModal`
- `useAccountModal`
- `useChainModal`

這些鉤子中的每一個都返回一個對象，該對象帶有打開其相應模態的功能。 請注意，如果您的應用程序不處於模態打開所需的狀態，則返回的功能將是未定義的。

```tsx
import {
  useConnectModal,
  useAccountModal,
  useChainModal,
} from '@rainbow-me/rainbowkit';

export const YourApp = () => {
  const { openConnectModal } = useConnectModal();
  const { openAccountModal } = useAccountModal();
  const { openChainModal } = useChainModal();

  return (
    <>
      {openConnectModal && (
        <button onClick={openConnectModal} type="button">
          Open Connect Modal
        </button>
      )}

      {openAccountModal && (
        <button onClick={openAccountModal} type="button">
          Open Account Modal
        </button>
      )}

      {openChainModal && (
        <button onClick={openChainModal} type="button">
          Open Chain Modal
        </button>
      )}
    </>
  );
};
```

每個鉤子還返回一個布爾值來表示模態的狀態。 通常建議您純粹依靠 Wagmi 鉤子（即 `useAccount`）直接對用戶的錢包連接狀態做出反應，而不是依靠連接模態的狀態。

```tsx
const { connectModalOpen } = useConnectModal();
const { accountModalOpen } = useAccountModal();
const { chainModalOpen } = useChainModal();
```

RainbowKit 設計為非中斷性和響應性，因此 dApps 應始終為用戶顯示界面，無論他們是否已連接錢包。 用戶可以直接從 MetaMask 連接或斷開他們的錢包，因此 dApp 必須直接響應基礎連接狀態。

依賴於 Ethereum 登入驗證機制的 dApps 應改為依賴 [身份驗證](/docs/authentication) 功能。
